Overview

This project was designed for my Microinteractions class at Drexel University. In the project I was assigned the task to create meaningful microinteractions of our liking. We did research on some Microinteractions we liked and deconstructed them to understand what engages the user and through what ways. I did research on creating a hover card that displays basic details of the project on hovering with the cursor. The details card also has an option to view a full preview of the project. Once the ‘View Project Preview’ button is clicked, a modal opens which shows the hero image of the project and other main details like Project Info, Roles & Responsibilities, and tools used.

Context and Challenge

As mentioned before, this project was done for my Microinteractions class and was supposed to be completed in a 10 week period of time. The languages we used for the project were HTML, CSS, and Javascript. Ultimately, I wanted to use this class to design something that I could use in my portfolio as well. I had a lot of projects in my portfolio and wanted to do something that lets a user take a quick overview of my project without actually opening the whole project. This led me to search about the kind of microinteractions that enhance user experience when looking through multiple projects online.

The Problem

The problem was to find a microinteraction that is both meaningful and useful in such a way that it can be applied to my online portfolio. I wanted something that gives surface level details of the project so that the visitor can look into my work without opening the full webpage. The main problem was to use code that is both simple and yet provides a good user experience.

DESIGN GOALS

My goal was to create a preview card and a preview modal. The preview card would show up if someone hovers their mouse or pointer on one of my project cards. The preview card would show details such as Title, Summary, and a View Project Preview Button, which opens the preview modal when it is clicked. The preview modal would have details such as The Hero Image, Project Info, Roles and Responsibility and tools used. This modal would give the user a high-level overview of my projects.

Tangible Goals & Metrics Used

Some of the tangible goals would be to create a hover effect on my project card that shows the preview card. The other part would be the creation of the preview modal and linking it to the preview card. I would consider this project successful if I can implement both of these parts and if it provides a good user experience to other users that will be testing the project. I would also create a google form survey that would help me with feedback once I finish the project.

PROJECT CHALLENGES:

Facing the problem of transitions: Once I was able to set all the code for the Preview Card and the Preview Modal. I realized that I was not able to make the details of the preview card appear one after the other. I used multiple ways to add a transition delay to the sub elements of the Preview Card but almost all of them were unsuccessful. I even tried to book two meetings with my professor but every code I used broke the previous code I had. Similar things happened when I tried to animate the opening and closing of the Preview Modal. I found that the code used for animating the opening and closing of the Modal used SASS which I had little to no previous experience with. I also discussed it with my professor about the transitions and animations but little progress was achieved in these two tasks.I also realized that creating transition delays in Preview Card does not add much effect on the user experience. Ultimately, I decided to skip working on the code for transition delays in Preview Card and animating the Preview Modal. This was necessary as I had multiple projects and very little time.

RESULTS

The results were not perfect but satisfying. I think the project was a success as I was able to create both the Preview Card and Preview Modal and both of them functioned well as well as showed all the details I wanted to show through them, which were my direct goals of this project.

Through my surveys I found 75% of the people liked the Preview Card and Modal and liked how it offered a short overview of my projects without getting too deep in the project.

Even Though I could not work on transition delays and modal animations I still learned a lot in this class through this project. I realized the importance of setting a realistic goal in the beginning and working on small elements of the microinteractions one by one. I also learned a lot about reusing code from already made examples and other websites and how to tailor it so it fits in my webpage.

In the future, I would love to add more transition delays and nice animations to every small element of the microinteraction I created.
